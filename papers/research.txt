IQRA'EVAL — RESEARCH PLAN OVERVIEW
====================================

Goal: Competitive submission to the next IqraEval shared task
      (Qur'anic Mispronunciation Detection & Diagnosis)

Hardware: 1x A100 40GB (burst to 2x)
Backbones: wav2vec2-bert / XLS-R-1B AND whisper-large-v3 (ablate both)
Framework: PyTorch + HuggingFace Transformers + s3prl (optional)


EXECUTION ORDER
===============

  step1.md  — Environment setup, data download, project scaffold, eval script
  step2.md  — Frozen SSL + CTC head (minimal working baseline)
  step3.md  — Full fine-tuning, single stage (wav2vec2-xls-r + whisper-large-v3)
  step4.md  — Multi-stage domain adaptation
  step5.md  — Synthetic mispronunciation augmentation
  step6.md  — Full solution: combine best findings + decoding + ensemble

Each step is self-contained and builds on the previous one.
Code from earlier steps is reused — later steps only add/modify.


METRICS (from IqraEval evaluation protocol)
============================================

Primary:   F1-score (harmonic mean of Precision and Recall on mispronunciation detection)
Secondary: Precision, Recall, Correct Rate, Accuracy
Detail:    FRR (False Reject Rate), FAR (False Accept Rate), DER (Diagnosis Error Rate)
Alignment: 3-way alignment of (canonical, verbatim, predicted) phoneme sequences
